STEP 07

The Battle

Needs Polish

The full testroom battle engine runs with boss mode enabled. The player's 3 ghosts face the boss + minion team. Dice roll, abilities fire, resources accumulate, KOs happen. The boss HP bar at the top drains in real-time. Boss AI decisions are driven by personality type:

TYRANT
Commits all resources. Crushing Blow on triples chips sideline. Gets Sacred Fire on KOs. Retreats behind minions on phase shift.
TRICKSTER
Mirror Dice swaps dice with player. Steals 1-2 resources on win. Splits into illusions on phase shift. Swap to minion every 4 rounds.
SWARM
+1 die per living minion. Spawns new minions on cycle. Sacrifices weakest to negate big hits. Brood Burst on phase shift deals 2 per minion.
GLACIER
Frost Aura: 1 cold damage/round. Frozen Dice locks player's highest die to 1. Permafrost: caps player damage at 2-3. Absolute Zero on phase shift.
ENRAGE SYSTEM
After each player finishes (all 3 ghosts KO'd or boss team wiped for their round), the boss gains +1 Enrage. At enrage 3: +1 die. At enrage 5: +1 damage. At enrage 7: +1 more die. At enrage 9: DOUBLE all damage. This creates a natural timer -- later raiders face a much harder boss, but the HP pool has been chipped down by earlier players.

Design Notes

  • The testroom battle engine handles all this natively -- raid mode just overlays the boss HP bar
  • Spectators see a simplified feed via Firebase battleState snapshots
  • NEEDS POLISH: spectator view is barebones -- dice visuals, ability callouts, and drama are missing
  • NEEDS POLISH: phase transition splash should interrupt the battle visually, not just overlay
  • Boss AI is fully functional -- personality-specific pre-roll, post-roll, and damage modifiers all work
  • HP scaling is sub-linear: 2 players = 1.5x, 5 players = 3x, 10 players = 4x -- prevents grind

Working

AUDIT SUMMARY

System Status

Full audit of the raid system -- what's solid, what needs work, and what's missing entirely.

Working End-to-End

  • Spirit World book UI with 5 zone tabs, boss cards, lock/unlock state
  • Queue system with Firebase real-time sync and transaction-based instance creation
  • Boss HP scaling based on player count (sub-linear: 1x to 4x)
  • 4 boss personality types with full AI (Tyrant, Trickster, Swarm, Glacier)
  • Phase transition system (4 phases with unique boss dialogue and effects)
  • Enrage system (progressive difficulty for later raiders)
  • Badge award system with composite badge detection
  • Progression gating (badges unlock bosses in later tiers)
  • Raid points economy with shop (Spirit Packs, Premium Packs, Legendary Packs)
  • Testroom redirect with full boss mode parameters via URL
  • Heartbeat system for disconnect detection (30s timeout)
  • Stale instance cleanup (1hr for instances, 30min for queues)
  • Result screen with damage leaderboard, MVP, and killing blow credit
  • Spectator view with real-time battle state snapshots

Needs Polish

  • Queue-to-raid transition has no countdown animation or sound -- feels abrupt
  • Spectator battle feed is text-only -- needs dice visuals, HP bars, ability callouts
  • No victory celebration effects (confetti, sound, screen shake)
  • Raid points earned not shown on result screen
  • Art fallback is timber.jpg everywhere -- needs a generic "unknown" silhouette
  • No pre-raid strategy view (can't see other players' teams before start)
  • Badge award notification is invisible -- no popup or fanfare when badge is earned
  • Phase transition overlays don't interrupt the testroom cleanly
  • No sound design for any part of the raid flow

Untested / Broken

  • Full multi-player raid with 3+ real concurrent users (never tested at scale)
  • Disconnect recovery -- does the raid advance properly if a fighter drops mid-battle?
  • Race condition on boss defeat -- two players could drain boss to 0 simultaneously
  • Testroom boss mode integration -- does the URL redirect actually load correctly?
  • Battle state snapshot write frequency -- could overwhelm Firebase on rapid dice rolls
  • Minion wave before boss fight -- wave intro + battle + boss fight sequencing untested
  • 10-player full lobby performance (10 Firebase listeners, 10 heartbeats)

v0.76 AUDIT FIXES

Shadow Self Raid Audit

Full code-level audit of every raid code path. Traced queue flow, testroom integration, return flow, and multi-player sequencing.

Bugs Fixed

  • CRITICAL: Padded ghost damage inflation -- Bosses with fewer than 3 ghosts (e.g. Dark Fang solo) pad to 3 by duplicating the boss. KO'd padding was counted as damage dealt, inflating totals by 2-3x. Fixed: damage calcs now only count real boss ghosts via RAID_PARAMS.realBossGhostCount.
  • Double raid points -- checkRaidResult() called distributeRaidRewards() AND manually incremented points for the returning player. Killing-blow player earned 2x points. Fixed: removed inline increment, distributeRaidRewards handles all players.
  • HP display mismatch -- Queue view showed boss.baseHp (decorative field, e.g. 60) while the engine uses bossGhost.maxHp (per-fight HP, e.g. 9). Fixed: queue now shows bossGhost.maxHp to match actual gameplay.
  • Firebase listener leak -- cleanupRaid() had a TODO comment and never actually removed instance or queue listeners. Fixed: properly calls .off() on instance, battleState, and queue refs.

Verified Working

  • Queue flow: selectRaid() to joinRaid() to startRaidFromQueue() -- clean transaction-based instance creation
  • Testroom integration: URL params correctly inject boss ghosts into GHOSTS array, pad to 3, KO padding
  • Boss HP bar: tracks real damage per renderBattle cycle, drains pool correctly
  • Return flow: testroom redirects with damage/ghostsLost/result, multiplayer drains boss HP in Firebase
  • Multi-player sequencing: currentFighterIdx advances, next player's activeRaid listener triggers entry
  • activeRaid guard: raidResult param blocks re-entry, URL cleaned before initRaidSystem runs
  • Badge progression gating: composite badges auto-award when all prereqs are met
  • Disconnect detection: heartbeat + 30s timeout + transaction-based advancement

Balance Assessment

  • HP scaling is sub-linear (1x to 4x for 1-10 players). Each player fights a fresh copy of the boss ghost at its real maxHp. Total damage from all fights drains the shared pool.
  • Solo Dark Fang (pool 9, ghost 9 HP): ~60-40 player advantage. On target.
  • 2-player Mountain King (pool 45, ghost 30 HP): each player deals ~15-20 damage. Tight but winnable. On target.
  • 5-player Mountain King (pool 90, ghost 30 HP): each player needs ~18 avg damage. Challenging. On target.
  • Enrage system (+1 per fighter) keeps pressure on later fighters, preventing easy cleanup.
  • Minion damage counts toward boss pool drain -- all team damage is meaningful.
